# 架构说明

[English](ARCHITECTURE.md) | [中文](ARCHITECTURE_CN.md)

## 概览

DIT Pro 是基于 Tauri 2.0 的桌面应用，前端使用 React/TypeScript，后端使用 Rust。当前主要面向 macOS（Apple Silicon 原生），用于影视行业专业拷卡工作流。

```
┌──────────────────────────────────────┐
│             macOS 桌面               │
│  ┌────────────────────────────────┐  │
│  │    React + TypeScript 界面      │  │
│  │ （任务 / 卷宗 / 预设 / 报告 / 设置）│
│  └──────────┬─────────────────────┘  │
│             │ Tauri IPC（44 个命令） │
│  ┌──────────▼─────────────────────┐  │
│  │          Rust 后端             │  │
│  │  ┌─────────┐ ┌──────────────┐  │  │
│  │  │ 拷贝引擎 │ │ 哈希引擎      │  │  │
│  │  │         │ │ XXH64/XXH3/  │  │  │
│  │  │         │ │ XXH128/SHA/MD│  │  │
│  │  └────┬────┘ └──────┬───────┘  │  │
│  │       │             │          │  │
│  │  ┌────▼─────────────▼───────┐  │  │
│  │  │      工作流编排器         │  │  │
│  │  │    scan->copy->verify->MHL│  │  │
│  │  └──────────┬───────────────┘  │  │
│  │             │                  │  │
│  │  ┌──────────▼───────────────┐  │  │
│  │  │     SQLite（WAL 模式）   │  │  │
│  │  │   Checkpoint + ErrorLog  │  │  │
│  │  └──────────────────────────┘  │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

## Rust 模块

| 模块 | 类型 | 说明 |
|------|------|------|
| `copy_engine/` | 目录 | 多路径并行拷贝，4MB 原子写入缓冲 |
| `copy_engine/atomic_writer.rs` | 文件 | 原子 `.tmp` 写入 + 重命名，`Drop` 自动清理 |
| `hash_engine/` | 目录 | XXH64、XXH3、XXH128、SHA-256、MD5，多算法单次扫描 |
| `workflow/` | 目录 | 拷卡编排器：scan -> copy -> verify -> MHL -> notify |
| `mhl/` | 目录 | ASC MHL v2.0 XML 生成与链管理 |
| `checkpoint/` | 目录 | 崩溃恢复、任务状态持久化、恢复逻辑 |
| `volume/` | 目录 | 设备发现、设备类型识别、空间监控 |
| `io_scheduler/` | 目录 | 按设备并发控制与缓冲配置 |
| `db/` | 目录 | SQLite Schema、迁移、WAL 初始化 |
| `notify/` | 目录 | SMTP 邮件通知（成功/失败） |
| `config.rs` | 文件 | JSON 配置读写 |
| `preset.rs` | 文件 | 工作流预设 CRUD |
| `report.rs` | 文件 | 拍摄日报告与任务报告，HTML/TXT 渲染 |
| `tray.rs` | 文件 | 系统托盘（空闲/活跃/错误） |
| `error.rs` | 文件 | 33 个结构化错误码（E1001-E1704），8 类别 |
| `error_log.rs` | 文件 | 错误日志持久化到 SQLite |
| `debug_bundle.rs` | 文件 | 一键导出诊断 ZIP |
| `version.rs` | 文件 | 语义化版本、5 个发布通道、构建元数据 |
| `commands.rs` | 文件 | 44 个 Tauri IPC 命令处理器 |

## 拷卡流水线

```
1. 扫描源卡           枚举文件并统计总容量
        │
2. 冲突检测           对比目标路径，同大小文件做哈希比对
        │
3. 用户决策           逐文件：跳过 / 覆盖 / 保留两者
        │
4. 执行拷贝           并行写入所有目标
        │                原子 .tmp -> rename
        │                按设备并发调度
        │                每文件落盘检查点
        │
5. 哈希校验           重算目标哈希并与源哈希比对
        │
6. 生成 MHL           为每个目标生成 ASC MHL v2.0 清单
        │
7. 通知与收尾         发送邮件并更新系统托盘状态
```

## 错误码系统

| 范围 | 类别 | 说明 |
|------|------|------|
| E1000-E1099 | COPY | 文件拷贝操作 |
| E1100-E1199 | HASH | 哈希计算与校验 |
| E1200-E1299 | DB | 数据库操作 |
| E1300-E1399 | CONFIG | 配置管理 |
| E1400-E1499 | IO | 存储与文件系统 I/O |
| E1500-E1599 | MHL | ASC MHL 链操作 |
| E1600-E1699 | NETWORK | 邮件与网络 |
| E1700-E1799 | SYSTEM | 运行时与系统错误 |

严重等级：**Critical**（数据丢失风险） > **Error**（操作失败） > **Warning**（有问题但完成） > **Info**（通知）

## 版本管理

版本格式：`MAJOR.MINOR.PATCH[-prerelease][+buildmeta]`

| 通道 | 条件 | 用途 |
|------|------|------|
| Dev | `0.x` 且无预发布标签 | 本地开发 |
| Alpha | 预发布标签以 `alpha` 开头 | 内部测试 |
| Beta | 预发布标签以 `beta` 开头 | 扩大测试 |
| RC | 预发布标签以 `rc` 开头 | 发布候选 |
| Stable | `1.0.0+` 且无预发布标签 | 生产发布 |

构建元数据由 `build.rs` 注入：
- Git commit 短哈希（7 位）
- 构建日期（YYYYMMDD UTC）
- 预发布标签（通过 `DIT_PRE_RELEASE` 传入）
