# v1.0.0-rc.1 — 第一个候选版 / Release Candidate 1

## 新功能 | New Features

- **报告完整显示全部哈希算法**：HTML 报告和应用内报告都完整显示 5 种哈希（XXH64、SHA-256、MD5、XXH128、XXH3），并支持点击复制 / **Full hash algorithm support in reports**: all 5 hashes (XXH64, SHA-256, MD5, XXH128, XXH3) are now fully displayed in both HTML and in-app reports, with click-to-copy for every value
- **多目标速度显示**：传输速度现在能正确反映所有目标的总写入吞吐量，复制和校验阶段都适用 / **Multi-destination speed display**: transfer speed now correctly reflects total write throughput across all destinations in both copy and verify phases

## 问题修复 | Bug Fixes

- **卷空间叠加显示**：修复每次轮询都会重新生成 volume ID，导致空间显示叠加而不是刷新 / **Volume space stacking**: fixed volume IDs regenerating every poll cycle, which caused space display to stack instead of refresh
- **网络卷空间检测**：修复 SMB / NFS 挂载上 `statvfs` 返回异常值的问题，并增加 `df` 回退 / **Network volume space detection**: fixed garbage `statvfs` results on SMB and NFS mounts and added a `df` fallback
- **级联覆盖冲突**：修复级联模式在次级目标上不遵守 Overwrite 冲突策略的问题 / **Cascade Overwrite conflict**: fixed cascade mode not respecting the Overwrite conflict policy on secondary destinations
- **速度图阶段隔离**：图表现在只显示当前活跃阶段，不再出现复制与校验数据重叠 / **Speed chart phase isolation**: the chart now shows only the active phase instead of overlapping copy and verify data
- **设置事件监听器**：修复设置页切换时不稳定的事件监听行为 / **Settings event listener**: fixed the unstable listener behavior during Settings page changes
- **级联压力测试补齐**：新增 8 个覆盖级联场景的压力测试，包括单目标回退、主次目标流程、多目标一致性、损坏检测和 `.DS_Store` 过滤 / **Cascade stress tested**: added 8 stress tests covering cascade scenarios such as single-destination fallback, primary-to-secondary flow, multi-destination consistency, corruption detection, and `.DS_Store` filtering
