# 更新规划

[English](ROADMAP.md) | [中文](ROADMAP_CN.md)

本文档记录当前版本线的产品与工程更新规划。发布日志只记录已经交付的内容；这里记录尚未上线、需要设计和验证的工作。

## 当前结论

百度网盘同步、Google Drive 同步和通用 WebDAV/AList 同步不能直接按“拷卡流水线的最后一步”上线。云端是慢速、不稳定、可重试的网络存储，必须和本地拷卡解耦。

当前上线标准：

- 本地拷贝、校验、MHL 生成完成后，应尽快释放原素材盘。
- 云上传从已校验的目的盘读取，不能继续占用原素材盘。
- 云上传必须有独立队列、断点续传、暂停、取消、重试、限速和并发控制。
- 每个远端对象必须有明确的完整性状态：已哈希校验、仅大小校验、或需要回读复核。
- Proxy-only 上传要成为一等功能，用于快速发给剪辑、导演、DIT 或制片核对。

## P0：云上传队列重构

目标：把云同步从拷卡任务的附属步骤改成独立后台队列。

计划内容：

- 新增持久化上传任务：`cloud_upload_jobs`、`cloud_upload_items`、`cloud_upload_parts`、`cloud_upload_events`。
- 上传任务从本地已验证目标盘、Proxy 缓存或报告目录读取文件。
- 本地 offload 完成并通过校验后，即可卸载或释放源素材盘。
- 上传 worker 支持暂停、继续、取消、失败重试、指数退避、应用重启后恢复。
- 队列支持全局限速、单 provider 并发、单任务并发、磁盘读取并发限制。
- UI 提供上传队列视图，展示每个对象的上传、校验、失败与重试状态。

必须验证的行为：

- 一个云上传运行时，新建本地拷卡任务不应被阻塞。
- 一个云上传运行时，新建第二个云上传任务应进入队列或按并发策略执行。
- 上传过程中取消任务，应在当前分片边界内停止并保留可恢复状态。
- 应用退出或崩溃后重启，未完成上传任务可继续。

## P0：Proxy-only 快速审片交付

目标：把“只上传 Proxy”做成独立的交付模式，而不是完整云备份的附属选项。

上传模式：

- `只上传 Proxy`：只上传代理文件，优先用于快速审片与跨部门核对。
- `上传 Proxy + 报告`：上传代理文件、缩略图、Rushes Log、任务报告。
- `上传 MHL + 报告`：只上传校验资料，不上传素材或 Proxy。
- `完整云备份`：上传原素材、Proxy、MHL、报告和缩略图。

对象类型：

- `Original`：原素材。
- `Proxy`：代理文件。
- `MHL`：ASC MHL 清单与链文件。
- `RushesLog`：素材日志。
- `Thumbnail`：缩略图。
- `Report`：HTML/TXT/JSON 报告。

交互要求：

- Proxy-only 任务可以从已完成 offload 生成，也可以在 Proxy 生成完成后立即进入云上传队列。
- Proxy-only 不阻塞本地拷卡，不要求完整原素材云上传完成。
- 每个 Proxy 上传后仍要有完整性状态，不能只显示“已上传”。
- UI 需要清楚区分“审片交付”和“完整云备份”，避免用户误以为 Proxy-only 等同于素材归档。

## P0：云端完整性验证策略

目标：不同 provider 采用不同强度的校验，UI 必须诚实呈现校验等级。

通用策略：

- 本地为每个上传对象生成 manifest，至少包含路径、大小、mtime、SHA-256、MD5。
- 上传前记录源文件 hash；上传后记录远端 size、etag、provider hash、分片 hash 或回读 hash。
- 远端无法提供可信 hash 时，默认不能标记为“强校验通过”。

Google Drive：

- 使用 resumable upload。
- 上传完成后读取 file metadata。
- 可用时比对 `md5Checksum` 和 size；视频转码或 Google-native 文件不纳入强 hash 判断。

百度网盘：

- 使用官方分片上传流程：precreate、分片 upload、create。
- 本地记录 block md5 列表和整文件 md5。
- create 后比对远端 size、fs_id、md5 或 provider 可返回的等价字段。

WebDAV / AList：

- 只能假设基础能力为 write、read、list、stat。
- 上传后至少比对远端 size。
- 如果配置开启“回读复核”，下载远端对象流并重新计算 hash；否则标记为“仅大小校验”。

## P1：Provider 支持

- Google Drive：OAuth、resumable upload、metadata hash 校验、共享链接生成。
- 百度网盘：官方 API provider、分片上传、秒传能力检测、token 刷新。
- S3：multipart upload、etag 解释、可选 checksum header。
- WebDAV/AList：保持实验性支持，并在 UI 中标记校验能力限制。

## P1：队列与通知体验

- 上传队列支持按素材日、项目、目的地、对象类型筛选。
- 支持“上传完成后发送邮件/飞书/Slack 通知”。
- 支持生成只包含 Proxy 链接和报告链接的交付摘要。
- 支持凭据迁移到系统 Keychain，配置文件不保存明文 token。

## 验收测试清单

- 慢速 provider 模拟：限速、抖动、断连、超时、429/5xx 重试。
- 本地拷卡完成后释放源盘，云上传继续从目的盘读取。
- 云上传进行中添加新本地任务，本地任务可正常扫描、拷贝、校验、生成 MHL。
- 云上传进行中添加新云上传任务，队列状态和并发限制符合配置。
- Proxy-only 任务只上传 Proxy/报告类对象，不上传原素材。
- 校验失败触发重试；超过重试次数后状态为失败并保留诊断信息。
- 应用重启后恢复未完成上传，不重复上传已完成且校验通过的对象。
