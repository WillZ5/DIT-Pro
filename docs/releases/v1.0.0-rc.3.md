# v1.0.0-rc.3 — 复制流程安全加固 / Copy Pipeline Safety Hardening

## 关键修复 | Critical Fixes

- **[P0] 防止用户选择 Skip 的文件导致数据丢失**：用户跳过的文件现在不会参与后校验或自动重试；此前旧文件哈希不匹配可能触发重试清理并删除原目标文件 / **[P0] Prevent data loss on user-skipped files**: skipped files are now excluded from post-verification and auto-retry; previously, a hash mismatch on old files could trigger retry cleanup that deleted the original destination file
- **[P0] 级联模式次级任务同步**：当主目标被跳过或失败时，所有次级任务会正确标记为 `skipped`，避免误报校验失败和错误的任务状态 / **[P0] Cascade mode secondary task sync**: when the primary destination is skipped or fails, all secondary tasks are now marked as `skipped`, preventing false verification failures and incorrect job status

## 重要修复 | Important Fixes

- **[P1] 全面使用基于哈希的校验**：将 `SkipIfSameSize` 替换为 `SkipIfVerified`（XXH3-128 全内容哈希），冲突判定不再信任文件大小 / **[P1] Hash-only verification everywhere**: replaced `SkipIfSameSize` with `SkipIfVerified` (full-content XXH3-128 hash); file size is no longer trusted for conflict resolution
- **[P1] `ConflictAction::Skip` 数据库一致性**：在标准、级联主目标和级联次目标路径下都会写入 `status='skipped'`，不再留下僵尸 `pending` 任务 / **[P1] `ConflictAction::Skip` DB consistency**: Skip now writes `status='skipped'` in standard, cascade-primary, and cascade-secondary paths, eliminating zombie `pending` tasks
- **[P1] 恢复路径逃逸保护**：源路径不匹配的任务会标记为 `failed`，不再静默跳过；收尾阶段会检查 `pending==0` / **[P1] Resume path-escape protection**: mismatched source paths are now marked as `failed` instead of being silently skipped, and finalization now checks `pending==0`
- **[P1] 并发恢复竞争条件**：`resume_offload` 增加 `contains_key` 保护 / **[P1] Concurrent resume race condition**: `resume_offload` is now guarded by a `contains_key` check
- **[P1] 崩溃恢复 UI**：仅在断电等异常中断后显示橙色 “Recover” 按钮，不再出现含糊的 “Continue” / **[P1] Crash recovery UI**: now shows the orange "Recover" button only after power loss, instead of the ambiguous "Continue" button

## 其他修复 | Other Fixes

- **[P2] 哈希回退查询**：检查全部 5 个哈希列（XXH64 / SHA256 / MD5 / XXH128 / XXH3），不再只查 XXH64 / **[P2] Hash fallback query**: now checks all 5 hash columns (XXH64 / SHA256 / MD5 / XXH128 / XXH3), not just XXH64
- **[P2] 自动重试遵守 `max_retries`**：超过限制的任务会保留为失败并给出警告 / **[P2] Auto-retry respects `max_retries`**: tasks over the limit remain failed with a warning
- **[P2] 统计与报告**：`skipped` 任务计入完成总数，保证进度显示一致 / **[P2] Stats and reports**: `skipped` tasks now count toward completed totals for consistent progress display
- **[P2] 校验失败计数修正**：按目标数量维度统计，不再错误使用源文件数 / **[P2] Verify failed count**: now uses the destination-count dimension instead of source file count
- **[P3] 删除死代码**：移除 `start_offload` 的并发守卫，新的 UUID 不可能冲突 / **[P3] Removed dead code**: dropped the `start_offload` concurrent guard because a fresh UUID cannot collide
