# v1.2.0-rc.1 — 首个候选版：通知、系统通知与级联测速 / First Release Candidate: Sounds, Notifications, and Cascade Benchmarking

v1.2 的首个候选版本，新增三项核心能力 / The first release candidate for v1.2, introducing three major capabilities.

## 新功能 | New Features

### A. 通知声音（P0） | Notification Sounds (P0)
- 为 4 类事件新增 Web Audio 提示音：任务完成、任务失败、源卡可弹出、警告 / Added Web Audio chimes for 4 event types: task complete, task failed, source released, and warning
- 设置页支持按事件启用或禁用，并提供音量滑块 / Added per-event enable or disable controls and a volume slider in Settings
- 每个提示音都带有测试预览按钮 / Added a preview button for each chime
- 新增工具文件 `src/utils/audioChimes.ts`，替换 `JobsView` 中的内联 `AudioContext` 代码 / Added `src/utils/audioChimes.ts` and replaced the inline `AudioContext` logic in `JobsView`

### B. macOS / Windows 系统通知 | macOS / Windows Push Notifications
- 集成 `tauri-plugin-notification` 并加入权限请求流程 / Integrated `tauri-plugin-notification` with a permission request flow
- 支持按事件控制通知：任务完成、任务失败、源卡可弹出 / Added per-event notification toggles for task complete, task failed, and source released
- 设置页新增通知权限状态指示 / Added a permission status indicator in Settings
- 新增工具文件 `src/utils/notifications.ts` / Added `src/utils/notifications.ts`

### C. 级联测速策略 | Cascade Benchmark Strategy
- 使用 64 MB 顺序写入测速 `benchmark_write_speed()` 替代基于设备类型的启发式判断 / Replaced the device-type heuristic with a 64 MB sequential write benchmark using `benchmark_write_speed()`
- 新增 Tauri 命令 `benchmark_dest_speed` / Added the `benchmark_dest_speed` Tauri command
- 每个目标路径旁显示实测速度 / Displayed the measured speed next to each destination path
- 手动重排目标顺序时会自动关闭速度优先 / Automatically disabled speed priority when destinations are reordered manually
- 设置页新增级联策略选择：速度优先或自定义顺序 / Added a cascade strategy selector in Settings: Speed Priority or Custom Order
- `OffloadDefaults` 新增 `cascade_strategy` 字段 / Added a `cascade_strategy` field to `OffloadDefaults`
